Project Roadmap
🗄️
In Progress
Phase 01 — Schema Design

Normalised PostgreSQL schema with carefully scoped relationships between venues, events, seats, and booking records.

  • Users, Events & Venue tables
  • Seat inventory modelling
  • Booking & reservation schema
  • Migration management
🔒
In Progress
Phase 02 — Concurrency & Locking

Prevents double-booking under load using PostgreSQL SELECT FOR UPDATE and advisory locks with full async execution.

  • Pessimistic row-level locking
  • Advisory lock strategies
  • asyncpg connection pooling
  • Race condition test suite
🐳
Pending
Phase 03 — Containerization

Production-grade Docker setup with multi-stage builds, compose orchestration, and environment-aware config layers.

  • Multi-stage Dockerfile
  • docker-compose orchestration
  • Health checks & restart policy
  • Env-based secrets management
